# Quality & Compliance Scorecards

### One-line positioning

**Zero-math, manager-defined yes/no quality and compliance checks — AI-evaluated with evidence — so teams can enforce standards at scale without building a scoring system.**

---

# Executive summary

Call Score V2 replaces **configuration-heavy call scoring** with a simple philosophy:

A **Scorecard is a mandatory checklist.**

Managers define **what must happen in a call**.
AI evaluates each check as **Pass / Fail**.
The system then classifies the overall call outcome:

**Strong → Acceptable → Needs Attention → Non-Compliant**

Each evaluation includes:

• transcript evidence
• timestamps
• AI explanation

Scorecards are:

• **disabled by default**
• **admin-controlled**
• **limited to 20 checks**
• **fully separated from Coaching / Skills**

**Coaching improves skills.
Scorecards enforce standards.**

This makes Empower especially valuable for **Insurance and Recruitment teams** needing:

• consistent quality standards
• scalable QA monitoring
• clear compliance evidence
• minimal configuration effort

— without debates over weights, percentages, or scoring formulas.

# Scorecard Structure

Simple by design.

Each Scorecard contains:

• Name
• Assigned groups / users
• Mandatory checks

Nothing else.

No weight configuration.
No scoring formulas.

---

# Check Structure

Each check includes only:

• Title
• Prompt
• AI evaluation

Example checks:

**Compliance Disclosure Mentioned**

“Verify the mandatory disclosure or call recording notification was clearly stated.”

**No Guaranteed Claims**

“Ensure the agent did not promise guaranteed outcomes or misleading results.”

**Salary Expectations Captured**

“Confirm the agent asked and recorded candidate salary expectations.”

**Clear Next Step Confirmed**

“Verify the call ended with a clear agreed next action.”

# AI Evaluation Logic

Each check is evaluated as:

**Passed**
or
**Failed**

AI determines severity and classifies the call:

| Outcome         | Meaning           |
| --------------- | ----------------- |
| Strong          | All checks passed |
| Acceptable      | Minor issues      |
| Needs Attention | Multiple failures |
| Non-Compliant   | Serious violation |

The system shows:

• transcript snippet
• timestamp
• AI explanation

Managers **do not configure logic**.

# Operational Automation (NEW)

## Workflows After Call Evaluation

Once a call is evaluated by a Scorecard, Empower can trigger **automatic workflows**.

This turns Scorecards from **analysis → operational enforcement**.

---

### Non-Compliant Calls

Possible workflows:

• alert supervisor immediately
• flag for compliance review
• create QA investigation ticket
• assign mandatory coaching session

---

### Needs Attention Calls

Possible workflows:

• add call to QA review queue
• notify team manager
• assign coaching task
• include call in training review library

---

### Strong Calls

Possible workflows:

• add to Best Practice Library
• highlight in team training
• use as onboarding examples

---

### Why Workflows Matter

Scorecards do not just **identify problems**.

They allow teams to **automatically act on them**.

This is critical in industries where **risk and compliance cannot rely on manual monitoring**.
